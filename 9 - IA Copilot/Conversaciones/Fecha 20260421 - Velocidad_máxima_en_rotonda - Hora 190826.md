[Timestamp: 2026/04/21 19:14:55]

**ai**: Una API (Interfaz de Programación de Aplicaciones) proporciona un conjunto de reglas y protocolos que permiten que diferentes aplicaciones de software se comuniquen entre sí.

En esencia, una API proporciona:

1.  **Acceso a funcionalidades y datos**: Permite que una aplicación utilice las capacidades o la información de otra aplicación o servicio sin necesidad de entender cómo está implementada internamente. Por ejemplo, una API de mapas permite que tu aplicación muestre mapas sin tener que desarrollar tu propio sistema de mapeo.
2.  **Un contrato de comunicación**: Define cómo deben hacerse las solicitudes (qué datos enviar, en qué formato) y cómo se recibirán las respuestas (qué datos esperar, en qué formato). Esto asegura que las aplicaciones puedan "hablar" el mismo idioma.
3.  **Abstracción**: Oculta la complejidad subyacente del sistema que la proporciona. Los desarrolladores solo necesitan saber cómo usar la API, no cómo funciona el servicio en su totalidad.
4.  **Integración**: Facilita la conexión entre sistemas dispares, permitiendo que trabajen juntos de manera fluida. Esto es fundamental para construir ecosistemas de software complejos y servicios interconectados.
5.  **Seguridad y control**: Permite al proveedor de la API controlar qué funcionalidades y datos se exponen, a quién y bajo qué condiciones (por ejemplo, mediante claves de API o autenticación).

En resumen, una API actúa como un intermediario que permite que el software interactúe, comparta datos y extienda sus capacidades.
