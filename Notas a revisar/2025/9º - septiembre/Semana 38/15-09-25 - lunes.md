# SQL

## Caracteres Especiales

% Cualquier cadena de cero o más caracteres

> [!example]
> WHERE Titulo LIKE '%computadora%' busca todos los títulos de libros que contengan la palabra 'computadora' en cualquier parte del título.

_ (subrayado) Cualquier carácter individual.

> [!example]
> WHERE Sigla LIKE '_RLA' busca todas los siglas de cuatro letras que finalicen con RLA.

[ ] Cualquier carácter individual de intervalo ([a-f]) o del conjunto ([abcdef]) especificado

> [!example]
> WHERE Apellido LIKE '[C-P]arrizo' busca apellidos de autores que terminen con arrizo y comiencen con cualquier carácter individual entre C y P.

`ORDER BY SALARIO Desc , APYN Asc`

Ordena por `SALARIO` y en aquellos donde el salario sea igual los ordenada por el `APYN`

Puedo utilizar ciertas funciones de fabrica como `MAX` o `MIN`.

Proyectar el salario máximo entre todos los empleados:

```sql
SELECT MAX (SALARIO)
FROM EMPLEADO;

```

Si pusiera `SELECT MAX(SALARIO), SELECT MIN (SALARIO)` me ponda uno al lado del otro, porque creara una columna para cada uno.

`SELECT DISTINT` No me muestra los repetidos. Ahora el `SELECT COUNT(DISTINT.....)` da en una celda 1 x 1 .

`COUNT` cuenta.

`GROUPY BY` : Agrupa las filas en grupos no repetidos, si tengo `GROUPY BY GENERO` en vez de tener una gran cantidad de filas con femenino y masculino, este solo me devolvera dos filas, una de Femenino y otra de Masculino. A partir de ahi se trabaja con el grupo entero. Si pongo:

```sql
SELECT GENERO, COUNT(*) AS CONTADOR
FROM EMPLEADO
GROUP BY GENERO
-- Comentario
```

Me devolverá:

| GENERO    | CONTADOR |
| --------- | -------- |
| Femenino  | 15       |
| Masculino | 28       |
| NULL      | 2        |

En NULL se agrupan aquellos que no tienen nada en `GENERO`

`HAVING` Acompañado de una condición, filtra del `GROUPY BY` como si fuera un `WHERE`
