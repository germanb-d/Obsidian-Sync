# Normalización

Aunque dado los pasos de Conceptual - > Lógico - > Físico debería darnos una base ya normalizada, igualmente debe verificarse su normalización.

- Minimización de Redundancia (la redundancia provoca inconsistencias)
- Minimizar anomalías de modificación
- Evitar perdida de integridad de datos
Umplica una serie de revisiones llamada Formas, las mas importantes siendo las primeras 3.

## Anomalias

`Empleados = (<u>idempleado</u>, nombre, salario, fecha_ingreso, iddepto, nombre_dpto)`

Acá vemos que el `nombre_dpto` es innecesario si tengo el `iddepto`.

- Inserción (agregar 2 empleados del mismo depto - > repite el depto)
- Actualización (modificar el nombre de un dpto)
- Eliminación (eliminar un empleado - > borra el dpto)

## Dependencia Funcional

Es una restricción entre atributos.

Se dice que un atributo Y depende funcionalmente de un atributo X (denotado X  Y ), cuando para un valor dado de X siempre se encuentra el mismo valor para el atributo Y. Se debe notar que X e Y pueden representar, además, un conjunto de atributos.

Legajo - > Apellido (esta es una dependencia correcta, por cada legajo hay un apellido)

**Todas los atributos son dependientes de las claves**

### Dependencia Parcial

Un atributo B (no clave) depende de un subconjunto de A (clave).

### Dependencia Transitiva

Un atributo B (no clave) depende de un atributo C no clave.
