# Excepciones

Al ocurrir errores en los programas, se utiliza [[Excepciones]] para determinar que hacer ante un error.

En vez de crear `if` para cada caso que pudiera provocar un error.

- Separamos el manejo de errores del codigo normal
- Ver una Pila de Llamadas de los errores
- Podemos agrupar y diferenciar los errores

En [[Java]] se usa Exception y RuntimeException

Checked exception: Ocurren en tiempo de Compilacion (no permiten compilar ni ejecutar el codigo). En

Unchecked exception: Ocurren en tiempo de Ejecucion (ejecuta el codigo y salta error en la ejecucion, los errores en rojo en la consola)

Ambas [[Excepciones]] pueden estar escritas en el código [[Java]], la diferencia es q la chequeada te obligara a escribir un try catch para tratarla.

Ante una `Checked exception` hay dos opciones:

-  Try-catch: atrapar el error y ejecutar algún codigo
- Se puede delegar la excepción al metodo que lo llamo. Ejemplo si con el back llamo al front y en caso de saltar un error no quiero que se trate hasta que vuelva al back.

Throwable esta arriba de las [[Excepciones]]: exception extiende de Throwable

```java
main{
    try{
    // codigo que se ejecuta
    }catch(/*error q sale (puede ser chequeada o no) */)){
        // que hacer ante el error
    } catch(){
    }
finally {
    // codigo que siempre se ejecutara
    }
}
```

El orden de los catchs siempre deben ser de los mas especificos arriba hasta los menos especificos abajo. En caso de que un catch especifico `IOexception` lo atrape este no entrara a ningun otro catch `Exception` #Parcial

Throws: delega

Throw: excepcion propia

Todas las expeciones propias extienden de exsepcion para que sea chequeada
